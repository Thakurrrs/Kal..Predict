# Documentation Index

This folder contains all mandatory SDLC artifacts for the Kal..Predict project.

## Sections

- `governance` - SDLC process, ownership, and approvals.
- `product` - Scope, outcomes, and acceptance criteria.
- `architecture` - Decisions, boundaries, and data contracts.
- `engineering` - Integration specs and engineering constraints.
- `quality` - Test strategy and verification rules.
- `security` - Threat model and secrets handling policies.
- `operations` - Runbooks and incident management.
- `compliance` - Logging, auditability, and change governance.

## UI-Specific Documents

- `architecture/adr-0002-ui-readonly-dashboard.md` - Read-only dashboard architecture decision.
- `engineering/ui-api-spec.md` - Contract for `/api/ui/*` endpoints and `/api/trial/*` paper-only controls.
- `quality/ui-test-strategy.md` - UI/API quality gates and acceptance criteria.
- `quality/pre-key-phase2-evidence-pack.md` - Pre-key verification outputs, limitations, and blocked-by-key section.

## Phase 2 Operating Documents

- `operations/pre-key-phase2-status.md` - Explicit pre-key progress, blocked items, and unblock criteria.
- `operations/current-status.md` - Short handoff note to resume quickly in new sessions.
- `engineering/performance-budgets.md` - Pre-key API/UI/inference latency budgets and verification commands.
- `engineering/future-enhancements.md` - Deferred maturity add-ons tracked for later phases.

## Document Metadata Standard

Every document must include:

- Owner
- Reviewers
- Approver
- Version
- Last Updated
- Status (Draft, In Review, Approved)
