# Project Structure (Documentation-First)

This project is initialized using an SDLC-first approach.

## Top-Level

- `context.md` - Vision and strategic context.
- `PROJECT_STRUCTURE.md` - Structure definition and ownership model.
- `docs/` - All governance, product, architecture, quality, security, operations, and compliance documents.

## Docs Tree

- `docs/README.md`
- `docs/governance/`
- `docs/product/`
- `docs/architecture/`
- `docs/engineering/`
- `docs/quality/`
- `docs/security/`
- `docs/operations/`
- `docs/compliance/`

## SDLC Rule

No implementation starts until gate-critical documents are approved in `docs/`.
